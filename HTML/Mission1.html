Présentation de la Mission 1

Contexte

Dans la mission 1, nous avons donc découvert en premier lieu KDou, qui est une entreprise de vente en grosses quantités pour professionnels. Avec l'accès à leur base de données, nous devions formuler avec l'aide des propriétés sélectives d'Excel plusieurs requêtes afin de filtrer les clients, produits, fournisseurs, etc. En fonction des consignes et restrictions appliquées.

C'est donc avec notre simple logique, nos connaissances antérieures, mais à l'aide de la facilité d'utilisation et de l'intuitivité de Microsoft Excel que nous devions réaliser cette première mission, signant aussi le début des SAE de notre 1ère année.

Bilan

Ce début de SAE n'est pas très compliqué, en effet, les premières questions ne sont que compréhension du tableau Excel ; on nous demande donc d'identifier les clés primaires et étrangères.

Question 1

"Dans chaque onglet, identifiez quelles sont les colonnes correspondantes ou pouvant correspondre à des clés primaires."

>>>> Une clé primaire est la donnée qui permet d'identifier de manière unique un enregistrement dans une table. Les différentes clés primaires des différentes tables sont les suivantes : - Tout d’abord pour ce qui est de la table “Fournisseur”, la clé primaire est le N°Fournisseur. - Concernant ce qui est de la table “Produit”, la clé primaire est la Réf produit. - La table “Catégorie” a pour clé primaire le code catégorie. - Pour ce qui est de la table “Clientèle”, la clé primaire est le Code Client. - Enfin la table “Commande” a pour la clé primaire est le N°Commande.

Question 2

"Dans chaque onglet, identifiez quelles sont les colonnes correspondant à des clés étrangères."

>>>> La clé étrangère fait référence à la clé primaire d’une autre table. Il n’y a aucune clé étrangère dans la table “Fournissseur” ainsi que dans la table “Catégorie”. il n’y pas de clé qui fait référence à la clé primaire d’une autre table. - Dans la table “Produit”, le #N°Fournisseur et le #Code catégorie sont les clés étrangères - Dans la table “Commande”, le #Code client est la clé étrangère. - Dans la table “Détail Commande”, le #N°commande et la #Ref produit sont les clés étrangères.

Ces 2 premières questions étaient donc relativement simples, de la pure analyse de documents qui nous permet d'entrer dans la matière, passons désormais aux "vraies" questions, celles de la manipulation de la base de données.

Proposition d'améliorations et conclusion

Propositions d'améliorations

Dans notre mission 1, nous avons, pour répondre à la question 5 proposés nombreuses améliorations pour que KDou puisse bénéficier d'une base de données exploitable correctement et surtout sans erreur, les voicis :

  • Normaliser les données, c’est-à-dire adapter le même format d’écriture pour les données, les noms, les villes, les pays et les fonctions dans la base de données. Par exemple, pour les noms de villes, il est important de décider si elles seront toutes écrites en français ou dans leur langue locale. Il faudrait aussi enlever les espaces en début, milieu et fin de chaîne de caractères afin de garantir une saisie uniforme.
  • Nous avons également décelé une principale erreur commune aux documents de l’annexe qui sont les soucis de doublons dû aux problèmes d’orthographe avec par exemple Sao Paulo sans les accents, Versailles avec un ï, et le doublon des représentants dans l’annexe. Nous pourrons régler ce problème très facilement en changeant dans les feuilles de Propriétés les éléments distinctes. Aussi mettre ces données sur une seule ligne pourrait améliorer l’organisation de ces données.

Conclusion

En conclusion, nous avons pu constater avec cette SAE qu'en réalité, un logiciel comme Excel n'est pas réellement optimisé pour stocker de très grande bases de données comme celle de KDou, bien qu'il le soit pour des petites à moyenne quantités de données. Pour de telles quantités de données, il faudrait une application beaucoup plus optimisées, afin de manipuler avec beaucoup + de précisions les données et les besoins des clients, une application comme Microsoft Access serait donc parfait pour de tels besoins
